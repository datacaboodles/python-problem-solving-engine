# Problem: Positive, Negative, or Zero

## Step 1: Input / Output / Data Types
- Input: a single integer
- Output: string
- Data Types: int → str

## Step 2: Constraints & Assumptions
- Constraints: No specific constraints are mentioned in the problem.
- Assumptions: The input will always be a valid integer.

## Step 3: Decision Logic
- A number is considered positive if its value is greater than zero.
- A number is considered negative if its value is less than zero.
- If the number is neither greater than nor less than zero, it is classified as zero.

## Step 4: Edge Cases
- The input number is zero.
- The input number is a negative integer.

## Step 5: Sub-parts
- Accept the input number.
- Determine which category the number belongs to.
- Display the corresponding result.

## Step 6: Pseudocode
- Receive an integer number as input.
- Decide whether the number is positive, negative, or zero.
- Store the result as a text value.
- Display the result.
